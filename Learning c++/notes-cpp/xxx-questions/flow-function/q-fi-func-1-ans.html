Assume these function definitions

For the following questions, assume that the proper includes and function prototype declarations have been made before these function definitions. 
   //--------------------- bigger
   int bigger(int a, int b) {
       if (a > b) {
          result = a;
       }else{
          result = b;
       }
       return result;
   }

   //--------------------- sum
   int sum(int c) {
       int total = 0;
       for (int i=1; i<=c; i++) {
          total += i;
       }
       return total;
   }
   
   //-------------------- signum
   int signum(int a) {
       if (a < 0) {
           return -1;
       }else if (a == 0) {
           return 0;
       }else{
           return 1;
       }
   }

   //----------------------- shiftLeft
   void shiftLeft(int& a, int& b, int c) {
      a = b;
      b = c;
   }

Part 1 - What's the output

What is the output from each of these code fragments?
  1. _____________ cout << bigger(2, 1); Answer: 2 [maximum of the values]
  2. _____________ cout << bigger(bigger(5,2), bigger(-6,8)); Answer: The function calls are made from the inside out. C++ doesn't specify whether parameters are evaluated from left to right or right to left, but we'll assume it's left to right. So bigger(5,2) is called and returns the value 5, making the problem bigger(5, bigger(-6,8). Then the other inner call is made with a result of 8, making the problem equivalent to bigger(5,8), which results in 8.
  3. _____________ cout << sum(3) + sum(4); Answer: 16 [sum(3)=1+2+3=6 and sum(4)=1+2+3+4=10]
  4. _____________ cout << sum(sum(sum(2))); Answer: 21, evaluating from the inside out, sum(sum(sum(2))) becomes sum(sum(3)) becomes sum(6), which is 1+2+3+4+5+6
  5. _____________ cout << signum(6%5 - 8); Answer: signum(-7) is -1
  6. _____________ int x=1; int y=2; shiftLeft(x, y, x+y); cout << y; Answer: 3

Part 2 - Why are these function calls illegal?

Possible errors.
  1. Wrong number of parameters.
  2. Wrong paramter type(s).
  3. Cannot assign to reference parameter. Actual reference parameter not an l-value.
  4. Void function called where a value is required.
Answer each of these questions with one of the alternatives above.
  1. _____________ cout << bigger(sum(2), signum(bigger(-3))); Answer: The bigger(-3) call needs two parameters.
  2. _____________ int x=1; int y=2; cout << shiftLeft(x, y, 3); Answer: Using a void function where a value is required.
  3. _____________ int x=1; int y=2; shiftLeft(3, y, x); cout << x; Answer: 3 is not an l-value. Ie, you can't assign to 3.
  4. _____________ int x=1; int y=2; shiftLeft(x+1, y+1, 3+1); cout << x; Answer: x+1 and y+1 are not l-values. Ie, they can't be assigned to.
