  1. Overload += operator.. 
    floatVector v1;
floatVector v2;
. . .
v1 += v2;

    This copies the values of the second list (v2) onto the end of the first list (v1). The second list (v2) remains unchanged.

  2. Constructor. Write a constructor that takes one float as a parameter. This would create a floatVector with one initial element. 
    floatVector v3(3.14159);
  3. Write operator< to compare two floatVectors element by element. 
    floatVector v1;
floatVector v2;

if (v1 < v2) ...

    This returns true if the corresponding elements of v1 are less than the elements of v2 (eg, v1[i] < v2[i]). If v2.size() <= v1.size(), then the comparison is true if all elements of v1 are less than the corresponding elements in v2. If v2.size() > v1.size() then the comparison is false.

  4. Write add(). Vectors can be modified to make evaluation of postfix expressions relatively easy by "pushing" values on the end of the vector (ie, use it as a stack), and defining functions that do the operations with the top elements. To make "instruction chaining" easy, these functions should return a reference to the floatVector (in the same way that operator= does). For example, we could add these functions to floatVector. The arithmetic functions "pop" the back two elements, do the mathematical operation, then push the result on the back again. 
    floatVector& push(float x);
floatVector& add();
floatVector& sub();
floatVector& mul();
floatVector& div();

    To evaluate the postfix expression

        1.2 3.4 + 2.2 -

    Using these functions the following calls could be written in a straightforward manner.

    fv.push(1.2).push(3.4).add().push(2.2).sub();

    Finally we get to the problem. Write a floatVector member funcion add() as described above.
