I've taken the basic floatVector test file and rewritten it to test xvector<float> and xvector<string>. Test Program for xvector.
operator<< for xvector
You can declare it like below. It should have a different template class name because it isn't part of the xvector class. 
    template<class T2> friend ostream& operator<<(ostream& os, const xvector<T2>& fv);
Destructor automatically called for each element in delete [] ...
I gave some wrong info in class. You do NOT have to explicitly call the constructor for each element of the array when it is deleted. C++ does this automatically. Explicit deallocation of each element is necessary in C, not C++ (at least not in this case).

      • Continue extending floatVector. The problem of extending floatVector has been assigned in three stages.
        1. Write the member functions: back, clear, empty, front, and pop_back. They should do the same thing as in the corresponding STL vector class. This should be completed.
